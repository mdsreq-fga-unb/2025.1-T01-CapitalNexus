# Backlog Geral

O backlog geral contém as funcionalidades e melhorias planejadas para o software. Cada item é detalhado para garantir uma visão clara do que será desenvolvido ao longo do projeto. Esta é uma lista preliminar e deve sofrer alterações, sempre que necessário, ao longo do projeto.

Para a elicitação de requisitos, foi utilizada User Stories (Histórias de Usuários), seguindo o processo escolhido para o projeto.

## Lista do Baklog Geral

| ID   | Declaração                                                                 | Critérios de Aceitação                                                                                                                                        |
|------|-------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|
| US01 | Eu, como visitante, quero primeiro acessar a história da equipe para conhecê-la.            | - Deve listar a missão, valores e visão da equipe <br> - Deve listar patrocinadores e projeto atual. <br> - Deve utilizar a identidade visual da equipe   |
| US02 | Eu, como visitante, quero visualizar os projetos da equipe para ver o seu trabalho.          | - Deve listar todos os projetos da equipe <br> - Cada projeto deve conter nome, descrição e imagem <br> - Deve ser acessível via menu                   |
| US03 | Eu, como visitante, quero visualizar os prêmios da equipe para entender sua relevância.     | - Deve estar listado os principais prêmios da equipe <br> - Cada prêmio deve conter nome, ano e informações sobre       |
| US04 | Eu, como visitante, quero entrar em contato com a equipe para algum objetivo específico.    | - Deve haver formulário de contato, com campo para e-mail, assunto e mensagem <br> - Deve haver confirmação de envio <br> - Deve haver um termo de compartilhamento dos dados e um botão de aceitar <br> - A equipe deve receber o contato                                              |
| US05 | Eu, como membro do marketing, quero postar conteúdo para atrair pessoas para a plataforma.  | - Deve haver formulário de criação de conteúdo, com campo de título e corpo <br> - Conteúdo deve aparecer na área pública         |
| US06 | Eu, como membro do marketing, quero excluir conteúdo existente para remover conteúdo que não deve ficar disponível. | - Deve haver botão de exclusão <br> - A exclusão deve ser confirmada <br> - Conteúdo deve ser removido da área pública                                       |
| US07 | Eu, como membro do marketing, quero editar a área pública para mantê-la atualizada.               | - Deve haver editor acessível  <br> - Edição deve ser salva e refletida imediatamente |
| US08 | Eu, como capitão da equipe, quero cadastrar membros para liberar o acesso à área interna.    |- Deve haver formulário com campos obrigatórios (matrícula, nome, e-mail, núcleo e cargo) <br> - Membro deve receber acesso à área interna <br> - Dados devem ser persistidos    |
| US09 | Eu, como membro da gestão de pessoas, quero atualizar as informações, núcleos e cargos do membro para manter sempre atualizado.          | - Deve haver campos editáveis para matrícula, nome, e-mail, núcleo e cargo <br> - Alterações devem ser salvas imediatamente  |
| US10 | Eu, como membro, quero consultar a lista de membros para visualizar informações dos membros.   | - Lista deve exibir nome, núcleos e cargos <br> - Deve conter filtros e campo de busca por nome, cargo ou núcleo.                       |
| US11 | Eu, como capitão da equipe, quero excluir membro cadastrado para remover membro que saiu da equipe.   | - Deve haver botão de exclusão <br> - Deve haver confirmação da ação <br> - Membro deve ser removido da base de dados                                         |
| US12 | Eu, como gestor, quero cadastrar uma reunião para que os membros acessem informações sobre ela. | - Deve haver formulário com data, hora, local e objetivo <br> - Reunião deve aparecer na página inicial da área de gestão                           |
| US13 | Eu, como gestor, quero atualizar informações da reunião para modificar data ou adicionar a ata.  | - Deve haver botão de editar <br> - Deve ser possível alterar data, hora, local e objetivo <br> - Membros devem ver as atualizações                                     |
| US14 | Eu, como membro, quero ser notificado da data de uma reunião, para que eu não acabe esquecendo.  | - Sistema deve enviar notificação 2 dias antes da reunião e no dia da reunião <br> - Notificação deve conter data, hora e local|
| US15 | Eu, como gestor, quero fazer uma chamada de cada reunião, para computar a presença dos membros.    | - Deve haver uma interface para registrar presença <br> - Membros ausentes devem levar uma falta <br> - Lista deve ser acessível após reunião               |
| US16 | Eu, como membro, quero ser notificado que levei falta, para ficar ciente da situação. | - Sistema deve notificar a falta recebida <br> - Notificação deve conter a data da falta                          |
| US17 | Eu, como membro, quero justificar a minha falta, para explicar minha ausência.   | - Deve haver formulário com campo para justificativa <br> - Deve ser possível anexar um arquivo de texto <br> - Deve ser enviado para a gestão de pessoas     |
| US18 | Eu, como membro da gestão de pessoas, quero avaliar a justificativa da falta para que o membro saiba se foi abonada ou não.                |  - Dado que sou gestor logado, quando/se acesso o módulo de faltas, então devo ver uma lista com: Nome do membro, data da falta e jusificativa enviada <br> Botão para aprovar ou não justificativa de falta (caso tenha sido enviada) <br> - Dado que sou membro, quando eu logar no sistema, então devo receber uma notificação de que minha justificativa foi avaliada.          |
| US19 | Eu, como membro da gestão de pessoas, quero receber um alerta para membro em risco de expulsão para saber quando um membro estiver nessa situação. | - Dado que sou o membro da gestão de pessoas, quando acesso o dashboard geral, então devo ver um alerta próximo aos nomes dos membros que estão em riscco de expulsão <br> - Dado que sou membro, quando faço login no sistema, então devo visualizar na minha área uma sinalização clara de que estou em risco de atingir o limite de faltas ou advertências. |
| US20 | Eu, como gerente de projeto, quero criar uma lista de materiais da equipe para guardar essa informação.    | - Dado que sou gerente de projeto logado, quando acesso a seção de materiais, então devo visualizar uma interface que permita criar uma nova lista, preenchendo: nome da lista, descrição da lista, status da lista. <br> - Dado que sou gerente de projeto, quando adiciono materiais a uma lista, então cada material deve ter os seguintes campos obrigatórios: nome, quantidade, descrição. <br> - Dado que sou gerente de projeto, quando salvo uma nova lista de materiais, então a lista deve ser persistida no sistema, ficando disponível para futuras consultas e edições.   |
| US21 | Eu, como gerente, quero adicionar, modificar ou excluir um material para manter a lista atualizada.  | - Dado que sou gerente de projeto logado, quando acesso a lista de materiais e escolho a opção de adicionar novo material, então devo conseguir cadastrar um material informando: nome, quantidade, descrição. <br>- Dado que sou gerente de projeto visualizando a lista de materiais, quando escolho a opção de editar um material existente, então devo poder alterar: nome, quantidade, descrição e o status do material selecionado.   <br> - dado que sou gerente de projeto visualizando a lista de materiais, quando escolho a opção de excluir um material, então o material deve ser removido da lista e o sistema deve confirmar a exclusão com uma mensagem.
| US22 | Eu, como membro, quero visualizar a lista de todos os materiais da equipe para poder consultá-los. | - Dado que sou membro logado no sistema, quando acesso a seção de materiais, então devo conseguir visualizar a lista completa de materiais.   <br> - Dado que sou membro visualizando a lista de materiais, quando utilizo a funcionalidade de filtros ou busca, então devo conseguir filtrar ou buscar materiais com base nos seguintes critérios: nome, status.   |
| US23 | Eu, como membro, quero marcar que vou usar algum material disponível da lista para eu utilizar. | - Deve haver botão ou ação para reservar material por um dia <br> - Nome do membro e data de uso devem ser salvos <br> - Status do material deve refletir “em uso”                |
| US24 | Eu, como membro, quero solicitar a compra de materiais que não estejam na lista para que seja disponibilizado.   | - Deve haver formulário de solicitação com nome do material, quantidade e po motivo do pedido <br> - Deve aparecer uma confirmação visual de envio                        |
| US25 | Eu, como membro da gestão de pessoas, quero gerar um relatório sobre o membros, para avaliar a participação e situação geral.  | - Deve retornar uma porcentagem de faltas e advertências geral dos membros <br> - Deve ser exportável <br> - Deve permitir filtros por período e núcleo                    |
| US26 |US26- Eu, como gestor, quero gerar um relatório de núcleos, para avaliar o desempenho, mudanças e situações de cada núcleo. | - Deve mostrar desempenho, mudanças e situações <br> - Deve permitir exportação <br> - Deve ter filtros por data                          |
| US27 | Eu, como membro do adm-fin, quero gerar um relatório de materiais em falta e as solicitações dos membros para eu avaliar a necessidade de comprar. | - Deve listar materiais com baixa quantidade e mais utilizados <br> - Deve listar os materiais mais pedidos <br> - Deve permitir exportação                   |
